# Changelog — 2026-05-14

Lore tracker split into focused modules and sub-components (#67). Security & config hardening: fail-fast env validation, JWT secret enforcement, closed-by-default CORS (#91).

The format is inspired by [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

### Issues
#67, #91

## [Unreleased]

### Added (web)

- **`src/lib/lore-game-state.ts` (#67):** New module with pure, testable game-state helpers — `LORE_WIN` constant, `loreEventTypeLabel`, `checkWinCondition` (returns `'p1' | 'p2' | 'both' | null`), and `resolveWinnerId`.
- **`src/routes/matches/[id]/lore/LorePanel.svelte` (#67):** New sub-component for the per-player lore button strip (increment/decrement buttons, score, recent-delta indicator). All panel, divider, keyframe, and landscape-orientation CSS moved here from the page.
- **`src/routes/matches/[id]/lore/LoreCloseGameModal.svelte` (#67):** New sub-component for the "leave game" confirmation modal, built on `LoreModal.svelte`.
- **`src/routes/matches/[id]/lore/LoreGameOverModal.svelte` (#67):** New sub-component for the game-result display shown when a game is won.

### Changed (web)

- **`src/routes/matches/[id]/lore/+page.svelte` (#67):** Reduced from 1 151 to ~900 LOC. The page now wires together the extracted modules and sub-components; duplicate P1/P2 panel markup, inline modal markup, and the associated CSS were removed. No change in visible behaviour.
- **`src/lib/config.ts` (#91):** `apiUrl` is now resolved via `resolveApiUrl()`, which trims `VITE_API_URL`, falls back to the documented `/api` default when unset, and logs a clear `console.error` in production builds if the var is missing.
- **`src/env.d.ts` (#91):** Corrected the `ImportMetaEnv` declaration from `API_URL` to `VITE_API_URL` to match the variable actually used by `config.ts`.

### API

- **`src/config/env.config.ts` (#91):** New module. `validateEnv()` is wired into `ConfigModule.forRoot({ validate })` so a misconfigured deploy fails fast during `NestFactory.create()`. `resolveJwtSecret()` returns the configured secret, or a dev-only placeholder outside production.
- **`src/main.ts` (#91):** CORS is now closed by default in production — when `CORS_ALLOWED_ORIGINS` is unset, origins are mirrored only outside production (`!isProduction()`). `bootstrap()` now has a `.catch()` that logs a clear startup-failure message and exits with code 1.

### Security

- **JWT secret enforcement (#91):** The hardcoded `'dev-secret-change-me'` fallback in `auth/jwt.strategy.ts` and `auth/auth.module.ts` is replaced by `resolveJwtSecret()`. `validateEnv()` rejects a missing or placeholder `JWT_SECRET` in production, so the app refuses to start rather than running with a publicly-known secret.
- **CORS no longer fails open (#91):** In production an explicit `CORS_ALLOWED_ORIGINS` allowlist is required (enforced by `validateEnv()`); CORS no longer defaults to mirroring any origin.
- **Env validation (#91):** `PORT` is validated as an integer in range `1–65535` at startup with a clear error message.
