### Changed (web)

- **`Select.svelte` — keyboard navigation:** Implemented the WAI-ARIA Select-Only Combobox pattern. The trigger button now carries `role="combobox"`, `aria-controls` (pointing to the listbox), and `aria-activedescendant` (tracking the keyboard-highlighted option). Arrow keys open the list and navigate options; Home/End jump to the first/last option; Enter/Space confirm the highlighted choice; Escape closes and returns focus to the trigger; Tab closes the list and lets natural focus flow continue. Opening via mouse click also preselects the current value for immediate arrow-key navigation.

- **`Select.svelte` — option highlighting:** Added `.select-custom__option--active` style (outline + background) to visually distinguish the keyboard-active option from the mouse-hover state.

- **`DeckPickerModal.svelte`, `PlayerPickerModal.svelte`, `TournamentPickerModal.svelte` — list-item button labels:** "Select" buttons now carry descriptive `aria-label` attributes (e.g. `"Select Amethyst Dawn"`, `"Select no deck"`) so screen readers announce context without requiring surrounding text.

- **`PlayerPickerModal.svelte` — disabled button announcement:** When a player is already chosen as the other side, the disabled button now carries `aria-disabled` and an `aria-label` explaining why (e.g. `"Amethyst Dawn already selected as other player"`), replacing the unreliable `title` attribute.

### Added (web)

- **Modal live regions (`DeckPickerModal`, `PlayerPickerModal`, `TournamentPickerModal`):** Each modal now includes a visually-hidden `role="status" aria-live="polite"` region that announces loading progress, errors, and result counts to screen readers without disrupting sighted layout.

### Added

- **`apps/web/AGENTS.md` — enterprise UI guideline (#58):** Concise reference covering design tokens, all shared primitives (`AppButton`, `AppCard`, `AppBanner`, `StatusStateCard`, `Select`, `AppToast`, `AppBreadcrumb`), navigation config (`navConfig.ts`), loading/empty/error state requirements, form conventions, accessibility rules (focus trapping, scroll locking, contrast, live regions), responsive guidance, and a new-feature adoption checklist. Cross-referenced from `changelogs/AGENTS.md`.

### Added (web)

- **`PlayerSelect.svelte` — new inline player search component:** Extracted opponent search into a self-contained `$lib/components/player/PlayerSelect.svelte`. The component owns debounced API calls, shows up to 2 quick-select buttons for partial matches (with a "+N more — use picker" hint when the result set is larger), auto-selects on an exact name match, and surfaces `offerGuestCreate` as a bindable flag for the guest-creation flow. The parent `matches/new/+page.svelte` now uses the component via `bind:value`, `bind:searchText`, and `bind:offerGuestCreate`, removing ~80 lines of inline search logic.
