## [Unreleased]

### Added (web)

- **`apps/web/src/styles/tokens.css`:** New module containing the Google Fonts `@import` and the entire `:root {}` block of CSS custom properties (colors, spacing, radius, shadows, glass-morphism vars, transitions). No rules — pure token declarations with zero side effects.
- **`apps/web/src/styles/layout.css`:** New module containing global resets (`box-sizing`, `prefers-reduced-motion`), base `html`/`body`/`a` styles, the `.app` shell (incl. background image), full `.topbar` + desktop nav + mobile nav drawer + language/team panels, `.main`, `.page`, and page-level layout helpers.
- **`apps/web/src/styles/components.css`:** New module containing all UI primitives — filters, cards, buttons, inputs, `.skip-link`, spinner, loading skeletons, alerts, `.playercard`, `.matchcard`, tabs, install prompt, and all utility classes (`.stack`, `.row`, `.btn-row`, etc.).

### Changed (web)

- **`apps/web/src/app.css`:** Replaced 2 555-line monolith with a thin 3-line `@import` orchestrator. Import order is documented in a comment: `tokens → layout → components`. The `+layout.svelte` import (`import '../app.css'`) is unchanged.

### Notes

- The only substantive path change in #49: background image URL updated from `url('./images/Background.png')` → `url('../images/Background.png')` to reflect the new location of `layout.css` one directory level deeper (`src/styles/` instead of `src/`).
- Files touched by #49: `apps/web/src/app.css`, `apps/web/src/styles/tokens.css` (new), `apps/web/src/styles/layout.css` (new), `apps/web/src/styles/components.css` (new).

### Issues (#51)

#51 [web][P1] Define shared page header and filter/results summary pattern

## [Unreleased] — #51

### Added (web)

- **`apps/web/src/lib/PageHeader.svelte`:** New shared component implementing the standardized list-page header pattern. Accepts `title`, `level` (h2/h3), `resultSummary` (always-visible result count), `chips` (dismissible active-filter chip array), and an `actions` snippet (primary CTA buttons). Includes inline guidance for new list pages in the component's JSDoc.

### Changed (web)

- **`apps/web/src/routes/matches/+page.svelte`:** Replaced the ad-hoc `div.row.matches-header` block with `<PageHeader>`. Result count (`filterSummary`) is now shown inline with the title via `resultSummary`; removed the duplicate count from the filter panel footer. Removed the `summary` prop from `FilterCard` (count is no longer hidden inside the collapsed card).
- **`apps/web/src/routes/players/+page.svelte`:** Replaced the bespoke `div.page-header` + `players-page__header-main` + chip-row block with `<PageHeader>`. The team-filter chip is now driven by the `chips` prop and styled consistently with the global pattern. Removed `summary` from `FilterCard`. Dropped dead `players-page__header-main`, `players-page__chip-row`, `players-page__filter-chip`, and `players-page__filter-chip-dismiss` scoped styles.
- **`apps/web/src/styles/components.css`:** Added global styles for `page-header__main`, `page-header__count`, `page-header__chips`, `page-header__chip`, `page-header__chip-dismiss`, and `page-header__actions`.
- **`apps/web/src/styles/layout.css`:** Removed `.matches-header` (identical to `.page-header`; both pages now use the unified class via `PageHeader`).

### Notes (#51)

- New list pages should import `PageHeader` from `$lib/PageHeader.svelte` and follow the guidance in the component's JSDoc comment.
- Files touched: `apps/web/src/lib/PageHeader.svelte` (new), `apps/web/src/routes/matches/+page.svelte`, `apps/web/src/routes/players/+page.svelte`, `apps/web/src/styles/components.css`, `apps/web/src/styles/layout.css`.
