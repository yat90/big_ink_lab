## [Unreleased] — #53

### Added (web)

- **`apps/web/src/lib/toast.ts`:** New toast store with `success / info / warning / danger / dismiss / schedule`. `schedule()` persists a toast in `sessionStorage` for post-navigation display (call before `goto()`). The module-level JSDoc documents the full feedback taxonomy and usage rules for each pattern.
- **`apps/web/src/lib/AppToast.svelte`:** New toast UI component. Renders a fixed overlay stack (bottom-center on mobile above the nav bar, bottom-right on desktop). Uses two separate `aria-live` regions — `polite` for success/info/warning, `assertive` for danger — so screen readers announce toasts at the appropriate urgency. Reads pending `sessionStorage` toasts via `afterNavigate` to support post-navigation success messages. Dismiss button included on each toast.

### Changed (web)

- **`apps/web/src/routes/+layout.svelte`:** Added `<AppToast />` once, at the `div.app` level, so toasts are available on every authenticated page.
- **`apps/web/src/routes/players/new/+page.svelte`:** On successful player creation, calls `toast.schedule('Player created', 'success')` before `goto('/players')`. Users now see a confirmation toast on the players list.
- **`apps/web/src/routes/players/[id]/edit/+page.svelte`:** On successful save, calls `toast.schedule('Player saved', 'success')` before `goto('/players/{id}')`. Users now see a confirmation toast on the player detail page.
- **`apps/web/src/routes/decks/[id]/edit/+page.svelte`:** On successful save, calls `toast.schedule('Deck saved', 'success')` before `goto('/decks/{id}')`. Users now see a confirmation toast on the deck detail page.

### Notes (#53)

- Feedback taxonomy (defined in `toast.ts` JSDoc): **toast** = transient success/non-blocking info; **AppBanner** = persistent inline error requiring action; **inline/field-level** = native validation or `.hint` span next to the field.
- Files touched: `apps/web/src/lib/toast.ts` (new), `apps/web/src/lib/AppToast.svelte` (new), `apps/web/src/routes/+layout.svelte`, `apps/web/src/routes/players/new/+page.svelte`, `apps/web/src/routes/players/[id]/edit/+page.svelte`, `apps/web/src/routes/decks/[id]/edit/+page.svelte`.
