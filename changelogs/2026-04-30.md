# Changelog — 2026-04-30

Staged changes snapshot: dashboard API, cached session, home data loading, nav/a11y, shared errors, list/match UI, new-match opponent flow, shared **deck-ink** constants (`packages/deck-ink`), deck picker / deck color UX on mobile, and mobile **More** menu layout plus logout confirm dialog.

The format is inspired by [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

### Issues

- **#30** — User-facing copy and helpers for failed HTTP requests (`apps/web/src/lib/errors.ts`): shared `ERR` strings, status mapping, and `messageFromFailedResponse`. Consumed by home dashboard load, lore quick-create match, matches list, decks list, players list, and player picker load paths in this batch.
- **#41** — Centralized Lorcana two-ink labels and ink order in **`deck-ink`**; **`DeckColorSelect`** portals the list to **`document.body`**, uses **fixed** positioning and flip/near-bottom logic so options stay above the mobile bottom tab bar; API/schemas consume the same option list and slug map.

## [Unreleased]

### API

- **Dashboard module** (`apps/api/src/dashboard/`): registered in `app.module`. New **JWT-protected** `GET /dashboard/summary?day=YYYY-MM-DD` returns a single payload: global **totals** (match count, game count, player count, deck count from existing services), **5 newest matches**, and **tournament dashboard window** (past / upcoming rows) for the given calendar day.
- **`deck-ink` workspace package** (`packages/deck-ink/`): canonical **`INK_ORDER`**, **`DECK_COLOR_OPTIONS`**, **`DEFAULT_DECK_COLOR`**, **`INK_CHART_COLORS`**, **`INK_SLUG_TO_CANONICAL`**, **`parseDeckColorLabel`**, runtime check that pair labels match **`INK_ORDER`**. **API** depends on **`deck-ink`**; **`matches/schemas/deck-color.enum.ts`** re-exports types and values (replacing the string enum); **`deck.constants.ts`** keeps only **`LINE_REGEX`**; **analytics**, **decks**, **matches**, **lorcana-match** / **deck** schemas, and **duels-replay.parser** import **`DECK_COLOR_OPTIONS`** / slug map where needed.

### Added (web)

- **`$lib/me.ts`:** `authMe` store and `setAuthMe` for the signed-in user + linked player payload (shape aligned with `GET /auth/me`).
- **`$lib/errors.ts`:** Shared user-facing `ERR` strings, `messageFromHttpStatus`, and `messageFromFailedResponse` for failed `fetch` / JSON error bodies — see **#30** above.
- **New match (`/matches/new`):** When search returns no players for the typed name, a **Create guest player** action posts a guest profile for that name and selects them as Player 2, with short copy about guest profiles and roster visibility.
- **`$lib/portal.ts`:** Svelte **`use:portal`** action to attach nodes to **`document.body`** (used by **`DeckColorSelect`**).
- **`a11y` (`focusTrap`):** Optional **`focusRoot`** — focus the trap container first ( **`tabindex="-1"`** ) so the first filter input is not auto-focused on open.

### Changed (web)

- **Root layout (`+layout.svelte`):** Populates or clears `authMe` on session verify, logout, missing token, and failed verify; **`<main>`** gets a dynamic **`aria-label`** (includes page title when not the default).
- **Home (`+page.svelte`):** Loads the dashboard from **`/dashboard/summary`** instead of five separate calls (`/matches/stats`, paginated players/decks, matches list, tournaments window). Subscribes to **`authMe`** for `myPlayerId`. **Lore** quick-create and dashboard errors use **`errors.ts`**. Network failures use **`ERR.network`**.
- **Desktop nav (`DesktopNavBar.svelte` + `app.css`):** **Statistics** uses an explicit **dropdown toggle** (expand/collapse, click-outside to close, route change closes). Dropdown panels open on **hover/focus only when `hover: hover`**; the **`.desktop-nav__dropdown--open`** class keeps the panel open for pointer/touch without hover.
- **`DeckPickerModal`:** Presets the player filter from **`get(authMe)`** instead of an async **`/auth/me`** fetch.
- **`PlayerPickerModal`:** Player load errors use **`messageFromFailedResponse`** / **`ERR.loadPlayers`**. Opening with **`presetTeamFromMe`** no longer performs a deferred **`/auth/me`** request — the modal becomes ready immediately; **the team filter is not auto-filled from the linked player** when `presetTeamFromMe` is true (previously it was set from `/auth/me`).
- **Players list (`players/+page.svelte`):** Load errors via **`errors.ts`**; default team filter uses **`get(authMe)`** instead of **`/auth/me`**.
- **Decks list (`decks/+page.svelte`):** Same error helpers; **Big Ink** default player filter uses **`authMe`** instead of **`/auth/me`**.
- **Matches list (`matches/+page.svelte`):** Load errors via **`errors.ts`**; initial player filter / “my matches” identity from **`authMe.subscribe`** (one-time apply) instead of **`/auth/me`**.
- **Stats (`stats/+page.svelte`):** Tournament fetch **`limit`** reduced from **200** to **100**.
- **New match (`matches/new/+page.svelte`):** Player 1 fixed to **linked user** via **`authMe`** (read-only **“Player 1 (you)”**). Player 2: **opponent name** field, debounced **player search** (`includeGuests`), **picker** button (**`IconUsers`**), removed inline **add player** form, stricter submit validation, **`PlayerPickerModal`** opponent-only settings (guest create when there is no match: see **Added** above).
- **`$lib/matches.ts`:** Re-exports **`DECK_COLOR_OPTIONS`** and **`parseDeckColorLabel`** from **`deck-ink`** ( **`svelte.config.js` `kit.alias`** maps **`deck-ink`** to package **source** for Vite).
- **`DeckColorSelect.svelte`:** Options list is **portaled**, **fixed**, **z-index** above the mobile **`.topbar`**, flips upward when space below is tight, respects **`#main`** scroll / **`visualViewport`**, **`max-height`** + scroll inside the list.
- **`DeckPickerModal.svelte`:** Filters use shared **`FilterCard`** (collapsed by default, badges, **`N deck(s)`** summary, clear name/color and restore preset player), **`filters__row`** layout aligned with decks/players lists; root **`focusTrap`** uses **`focusRoot`** on the modal card.
- **`DeckStatsContent.svelte`:** **`INK_CHART_COLORS`** and **`INKS`** from **`deck-ink`** instead of local constants.
- **`MobileNavBar.svelte`:** **Statistics** removed from the **mobile bottom pill** (still reachable under **More**).
- **`MobileNavDrawer.svelte`:** **Logout** moved above the spacer block (after **Players**); **Close menu** stays at the bottom; **logout** uses an in-app **confirm dialog** ( **`delete-game-modal`** pattern, **`z-index: 1300`**, **`Escape`** closes prompt before closing drawer).

### Fixed (web)

- **New match:** Avoid clearing Player 2 from debounced search when the field still matches the opponent chosen from the picker but multiple DB rows share the same name.
- **Deck color picker (mobile):** Dropdown no longer sits under the fixed bottom nav because it renders in a higher stacking context and measures **viewport** / **safe** bottom inset (**#41**).
