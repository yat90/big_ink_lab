# Changelog — 2026-05-16

Standardize loading, error, and empty states across web routes so every async fetch gives users visible feedback.

The format is inspired by [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed (web)

- **`me/+page.svelte`:** Replaced hard-coded English strings with i18n keys under `me.*`; loading state now uses `StatusStateCard kind="loading"` instead of a plain muted paragraph; save errors use `me.saveTeamError` / `me.savePlayerNameError` translation keys.
- **`decks/[id]/DeckViewMatchesTab.svelte`:** Loading, error, and empty states now use `StatusStateCard` (previously raw `<p>` elements with inline colour); added i18n keys `decks.matchesTab.*`.
- **`decks/+page.svelte`:** Player filter `<select>` is disabled with `aria-busy` while `fetchPlayers()` is in flight, preventing the dropdown from visibly popping from an empty to a filtered list.
- **`tournaments/results/+page.svelte`:** Tournament info fetch no longer silently falls back to a placeholder name on failure; errors now surface via `AppBanner variant="danger"` using the new `tournaments.results.tournamentLoadError` key.

### Fixed (web)

- **`decks/new/+page.svelte`:** `loadSuggestedName()` no longer swallows errors silently — failures are now shown as an `AppBanner variant="warning"` next to the "Another name" button (`decks.suggestError` i18n key).
- **`matches/new/+page.svelte`:** `fetchDeck()` no longer swallows fetch errors silently — failures surface as an `AppBanner variant="warning"` above the submit actions (`matches.new.deckFetchError` i18n key).
