## [Unreleased]

### Added (web)

- **`src/lib/players.ts` — shared `playerName` utility (#65):** Extracted the duplicated `playerName(p)` function into a single canonical module. The function resolves a `{ name?: string } | string | undefined` value to a display string, returning an en-dash `–` as the fallback. All five previous inline copies have been removed.

- **`src/lib/api-client.ts` — shared API fetch wrappers (#69):** New module with `getJson<T>`, `postJson<T>`, `patchJson<T>`, and `deleteNoContent`. Each wrapper prepends the API base URL, attaches the Bearer auth token when present, and throws a typed `ApiError` (carrying `status` and `message`) on non-2xx responses. `team.ts` fully migrated: the local `jsonOrThrow`/`noContentOrThrow` helpers removed, 13 functions simplified to single-line delegations.

- **`src/lib/storage.ts` — typed storage utility (#71):** New module with `getStorageItem<T>`, `setStorageItem<T>`, and `removeStorageItem` helpers. All use `JSON.parse`/`JSON.stringify` consistently and guard against SSR via `typeof window === 'undefined'`. All storage key constants consolidated into `STORAGE_KEYS`. Migrated four call sites: `auth.ts`, `i18n/index.ts`, `InstallPrompt.svelte`, and `lore/+page.svelte`.

- **`src/routes/matches/[id]/lore/+page.svelte` — `createCooldownState` factory (#72):** Replaced four per-player state variables (`p1CooldownFlash`, `p2CooldownFlash`, `p1CooldownFlashTimer`, `p2CooldownFlashTimer`) and the branching `flashCooldownIgnored()` function with a small `createCooldownState()` factory. Each player gets its own instance (`p1Cooldown`, `p2Cooldown`) exposing `.flash()`, `.active`, and `.dispose()`. Timer cleanup on component destroy now calls `dispose()` on each instance instead of duplicated `clearTimeout` guards.

- **`src/lib/matches.ts` — shared `gameWinnerId` utility (#66):** Extracted the duplicated `gameWinnerId(game)` function into `$lib/matches`. The function resolves a polymorphic `winner` field (`string | { _id: string } | undefined`) to a plain ID string. All six previous inline copies have been removed (five route files and `GameLine.svelte`).

- **`src/routes/+page.svelte`, `src/routes/matches/+page.svelte`, `src/routes/matches/[id]/+page.svelte`, `src/routes/matches/[id]/lore/+page.svelte`, `src/routes/tournaments/[id]/+page.svelte`:** Replaced local `playerName` definitions with an import from `$lib/players`. Unused `LorcanaMatchPlayer` type imports dropped from the three files where it was only referenced inside the now-removed function.

- **`src/routes/+page.svelte`, `src/routes/matches/+page.svelte`, `src/routes/matches/[id]/+page.svelte`, `src/routes/matches/[id]/lore/+page.svelte`, `src/routes/tournaments/[id]/+page.svelte`, `src/lib/components/match/GameLine.svelte`:** Replaced local `gameWinnerId` definitions with an import from `$lib/matches`. Redundant `g as Game` casts in the lore page removed.
